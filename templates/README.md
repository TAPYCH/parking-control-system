Smart Parking OS — это масштабируемое программное ядро для автоматизации парковочных пространств, контрольно-пропускных пунктов (КПП) и систем контроля управления доступом (СКУД). Платформа обеспечивает бесшовную интеграцию систем видеоаналитики, промышленных контроллеров, телефонии и учетных систем (1С) в единый центр управления.

✨ Ключевые возможности
🧠 Бизнес-логика и управление доступом
Гибкая политика доступа: Поддержка "белых" и "черных" списков, временных пропусков, гостевого доступа.

Адаптивное управление трафиком: Автоматическое сокращение времени бесплатной парковки при достижении порога загрузки (правило 80%).

Антифрод защита: Логика "Анти-паровозик" для предотвращения несанкционированного проезда двух транспортных средств вплотную.

Временной контроль: Настраиваемые таймеры (например, 2 часа бесплатно) с последующими уведомлениями или блокировкой выезда.

🔌 Поддержка оборудования (Hardware Abstraction Layer)
LPR/Видеоаналитика: Нативная интеграция с ПО "Линия" (Devline) и любыми IP-камерами, поддерживающими отправку HTTP-запросов.

Промышленные контроллеры: Унифицированные драйверы для управления реле Dingtian, Rodos, Kernel, а также поддержка IoT-реле Shelly, Sonoff.

Периферия:

Датчики: Обработка сигналов от фотоэлементов (ИК-барьеров) для детекции факта проезда.

Светофоры: Прямое управление светофорными объектами (Красный/Зеленый) на основе статуса проезда.

Табло: Вывод динамической информации для водителей на Web-мониторы и LED-панели.

🔗 Интеграции и API
Web-Интерфейсы:

Панель администратора: Функциональный Dashboard для мониторинга событий в реальном времени, настройки логики работы и управления номенклатурой оборудования.

Монитор водителя: Информационный экран для отображения статуса парковки и номера места.

REST API:

1С Интеграция: Предопределенные эндпоинты для двусторонней синхронизации базы номеров и выгрузки логов событий в учетную систему.

Сторонние сервисы: API для интеграции с системами распознавания и датчиками.

Коммуникации:

Телефония: Интеграция с Asterisk/FreePBX (функция "Открыть шлагбаум по звонку" для гостей).

Telegram Bot: Отправка уведомлений оператору о нештатных ситуациях, превышении лимитов или нарушениях.

🚀 Быстрый старт
Системные требования
ОС: Linux (рекомендуется Ubuntu 20.04+) / Windows Server

Python: 3.9 или выше

ОЗУ: от 2 ГБ

Сеть: Фиксированные IP-адреса для оборудования (камеры, реле)

Установка
Клонируйте репозиторий

bash
git clone https://github.com/ВАШ_НИК/smart-parking-os.git
cd smart-parking-os
Установите зависимости

bash
pip install -r requirements.txt
Настройка приложения

Первичная настройка сетевых адресов оборудования и параметров доступа производится через Web-интерфейс после запуска или в файле config.py.

Запуск сервера

bash
python main.py
После запуска сервис будет доступен по адресу http://localhost:8000.

Монитор водителя: http://localhost:8000/monitor

Панель администратора: http://localhost:8000/admin

📖 Документация API (v1)
1. Обработка событий от камер
Эндпоинт для приема данных с систем распознавания номеров (LPR).

http
GET /api/v1/camera/event
Параметры запроса:

Параметр	Тип	Обязательный	Описание
plate	string	Да	Распознанный государственный номер
direction	string	Да	Направление: in (въезд) или out (выезд)
gate_id	string	Нет	Идентификатор шлагбаума/ворот (по умолчанию: gate_1)
2. Обработка событий от датчиков
Эндпоинт для приема сигналов от ИК-барьеров и петлевых детекторов.

http
GET /api/v1/sensor/trigger
Параметры запроса:

Параметр	Тип	Обязательный	Описание
sensor_id	int	Да	Идентификатор датчика: 1 (въезд), 2 (центр), 3 (выезд)
3. Интеграция с 1С
Эндпоинты для взаимодействия с учетной системой.

Синхронизация базы номеров

http
POST /api/1c/sync_users
Body: JSON-массив объектов с полями plate, access_level, valid_until.

Выгрузка событий

http
GET /api/1c/get_logs
Возвращает: JSON-массив логов доступа за указанный период.

🏗 Архитектура решения
Проект построен на асинхронном фреймворке FastAPI, что обеспечивает высокую производительность при параллельной обработке запросов от оборудования.

main.py: Ядро системы. Инициализация приложения, маршрутизация (эндпоинты), Web-сервер.

hardware.py: Слой абстракции оборудования. Содержит драйверы для взаимодействия с реле, светофорами и датчиками различных производителей.

database.py: Модели данных и ORM (SQLAlchemy) для работы с встроенной базой SQLite (возможна замена на PostgreSQL).

templates/: Frontend-часть проекта (HTML, CSS, JavaScript) для отображения панелей мониторинга.
